Stage 1: Lexer / Tokenizer

The lexer reads raw source characters and groups them into tokens — the atomic units of meaning. It strips whitespace and comments, classifying each token by type.

How Lexing Works

The lexer uses a finite automaton — it scans character by character, transitioning states based on what it sees. When it reaches a state where no more characters can extend the current token, it emits the token and resets.


Token types in this grammar: keyword (int, if, return), identifier (names), number (integer literals), string ("..."), operator (+, -, *, /, =, ==, !=, <, >, <=, >=, &&, ||), punctuation (; , ( ) { }), invalid (unrecognized characters).

Stage 2: Parser & Abstract Syntax Tree

The parser takes the token stream and builds an Abstract Syntax Tree (AST) — a hierarchical representation of the program's structure. Each node represents a syntactic construct. Hover over a node to see what source text it corresponds to.

Stage 3: Intermediate Representation (IR)

The AST is lowered to 3-address code (also called IR or TAC). Each instruction has at most one operator and three operands. This form is architecture-independent and easy to optimize. Click "Next Step" to animate the translation one statement at a time.

About 3-Address Code

Every complex expression is broken into temporaries (t1, t2, ...). Each instruction does exactly one operation: t1 = a op b or x = t1. This regularity makes optimization straightforward — no deeply nested expressions to navigate.


Real compilers use SSA (Static Single Assignment) form where each variable is assigned exactly once, enabling powerful optimizations like dead code elimination and global value numbering.

Stage 4: Optimization & Code Generation

The optimizer transforms IR to be faster/smaller without changing semantics. Then the code generator maps IR to target machine instructions.

After optimization, the code generator maps IR instructions to real machine instructions. Each IR temp maps to a register or stack slot.

Instruction Legend

mov dest, src — copy value from src to dest register

imul dest, src — multiply dest by src, store in dest

add dest, src — add src to dest

cmp a, b — compare a and b, set CPU flags

jg label — jump if greater (based on flags from cmp)

mov [rbp-N], reg — store register to stack (local variable)

mov reg, [rbp-N] — load from stack into register
