tmpfs-academy

Hands-on Linux skills, learned in a real terminal — not a quiz

A web platform for practicing real system administration, application deployment, and troubleshooting. Instead of multiple-choice questions, learners are dropped into a live, disposable Linux environment and given a task to actually complete — validated against the real system state.

Linux open source in-browser + SSH early design

The Idea

Most "learn Linux" platforms test whether you can recall a fact. tmpfs-academy tests whether you can do the work. A learner goes from "visit the site" to "working in a shell" in seconds, with nothing to install. Every challenge is solved by performing real work on a real system — and success is determined by inspecting the actual machine state (files, services, processes, network), not by self-reporting.

Environments are sandboxed and ephemeral, so mistakes are expected and have no lasting consequences. Safe to break, objective to grade.

User Experience

Browse challenges. Pick one organized by theme (sysadmin, app deployment, troubleshooting) and difficulty.
Start a session. A fresh, isolated environment is provisioned on demand.
Connect. Work in the in-browser terminal, or over SSH from your own terminal using the credentials shown in the UI.
Solve the task. e.g. "the web app on port 8080 is down — bring it back up and make it survive a reboot." Investigate and fix it with standard Linux tooling.
Get graded. A validator checks the live system against the success criteria and reports pass/fail with actionable detail.
Session ends. The environment is torn down; progress is recorded against your account.

Challenge Themes

System Administration

Users and permissions, services, storage, networking, scheduled jobs, log analysis.

Application Deployment

Installing and configuring apps, reverse proxying, TLS, environment configuration, persistence across restarts.

Troubleshooting

Diagnosing broken services, resource exhaustion, misconfiguration, and connectivity failures from a degraded starting state.

Open Questions

  • Sandbox / isolation mechanism (containers vs. VMs vs. ephemeral hosts) and the security boundary it provides.
  • Browser terminal technology and the SSH gateway design.
  • Challenge definition format and the automated grading / validation model.
  • Authentication, persistence, and progress tracking.

Status
