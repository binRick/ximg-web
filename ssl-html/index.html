Certificate Chain Explanation

Root CA Certificate

Self-signed certificate stored in the browser/OS trust store. Issued by trusted Certificate Authorities (DigiCert, Let's Encrypt ISRG Root, Comodo, GlobalSign, etc.). The chain anchors here.

Intermediate CA Certificate

Signed by the Root CA. Used to issue end-entity certs. Limits exposure — the Root CA key can be kept offline. Servers must include intermediates in the TLS handshake (browsers may not have them cached). A server missing intermediates causes "incomplete chain" errors.

End-Entity (Leaf) Certificate

Issued to your domain. Contains: subject CN/SAN, public key, validity period, key usage extensions. Signed by the Intermediate CA. Your server presents this during TLS handshakes.

Chain Validation

Client walks up the chain: leaf → intermediate → root. At each step, the issuer's public key must verify the subject's signature. If any link breaks (expired, revoked, missing), the handshake fails with a certificate error.

Common Certificate Errors & Fixes

ErrorCauseFix
ERR_CERT_DATE_INVALID
CERT_HAS_EXPIRED		 Certificate past its NotAfter date, or system clock wrong Renew certificate. Check server/client system time is accurate (NTP). Let's Encrypt certs expire every 90 days — automate renewal with certbot.
ERR_CERT_COMMON_NAME_INVALID
SSL_ERROR_BAD_CERT_DOMAIN		 Certificate CN/SAN doesn't match hostname being connected to Issue cert with correct domain in SAN. Wildcard certs (*.example.com) only cover one level. Add www. and apex separately or use SAN list.
ERR_CERT_AUTHORITY_INVALID
SELF_SIGNED_CERT_IN_CHAIN		 Cert signed by untrusted CA, or self-signed cert presented to browser Use cert from trusted CA (Let's Encrypt is free). For internal services, install your internal CA cert in trust store.
ERR_SSL_PROTOCOL_ERROR
SSL_ERROR_RX_RECORD_TOO_LONG		 Plain HTTP served on HTTPS port, TLS version mismatch, firewall stripping data Ensure server is configured for TLS on port 443. Check nginx/Apache SSL config. Verify no proxy strips the TLS record.
UNABLE_TO_GET_ISSUER_CERT
UNABLE_TO_VERIFY_LEAF_SIGNATURE		 Incomplete certificate chain — intermediate CA missing from server config Configure server to serve the full chain (leaf + all intermediates). With nginx: use ssl_certificate fullchain.pem not just cert.pem.
ERR_CERT_REVOKED
OCSP_RESPONSE_STATUS_REVOKED		 Certificate was revoked by the issuing CA Issue a new certificate. Check why the cert was revoked (key compromise? CA decision?). Rotate any potentially compromised private keys.
ERR_SSL_VERSION_OR_CIPHER_MISMATCH Server only supports deprecated TLS versions or cipher suites the client won't use Enable TLS 1.2 and TLS 1.3 on server. Remove SSLv3, TLS 1.0, TLS 1.1 support. Enable modern ECDHE cipher suites.
NET::ERR_CERT_WEAK_SIGNATURE_ALGORITHM Certificate uses SHA-1 signature (deprecated) or MD5 Reissue cert with SHA-256 or better signature algorithm. Most modern CAs use SHA-256 by default.
