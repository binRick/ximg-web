Design Flow

1. RTL Design — Write Verilog or VHDL describing hardware behavior
2. Synthesis — Map RTL to FPGA primitives (LUTs, FFs, DSPs). Tools: Vivado, Quartus, Yosys
3. Implementation — Place & Route: assign logic to physical tiles, route interconnects
4. Timing Analysis — Static Timing Analysis ensures all paths meet setup/hold requirements
5. Bitstream Generation — Binary configuration file for the FPGA
6. Programming — Load bitstream via JTAG, SPI flash, or PCIe

Verilog Reference

Basic Module Template

module example (
  input  wire        clk,
  input  wire        rst_n,  // active-low reset
  input  wire [7:0]  data_in,
  output reg  [7:0]  data_out
);

always @(posedge clk or negedge rst_n) begin
  if (!rst_n)
    data_out <= 8'h00;
  else
    data_out <= data_in + 8'd1;
end

endmodule

4-bit Counter

module counter4(
  input  clk, rst,
  output reg [3:0] cnt
);
always @(posedge clk)
  if(rst) cnt <= 0;
  else    cnt <= cnt + 1;
endmodule

D Flip-Flop with Enable

module dffe(
  input      clk, en, d,
  output reg q
);
always @(posedge clk)
  if(en) q <= d;
endmodule

VHDL Reference

Basic Entity/Architecture

library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity counter is
  port(
    clk     : in  std_logic;
    rst     : in  std_logic;
    count   : out std_logic_vector(3 downto 0)
  );
end entity;

architecture rtl of counter is
  signal cnt : unsigned(3 downto 0);
begin
  process(clk)
  begin
    if rising_edge(clk) then
      if rst='1' then
        cnt <= (others=>'0');
      else
        cnt <= cnt + 1;
      end if;
    end if;
  end process;
  count <= std_logic_vector(cnt);
end architecture;
