You're right — esptool.py handles the flashing. The pipeline is:

arduino-cli compiles .ino → binary → esptool.py flashes it.

Wrote flash-sketch-001-button-beep.sh which automatically:

  • Installs arduino-cli via Homebrew if missing
  • Adds Espressif board index and installs ESP32 core
  • Installs TFT_eSPI + FT6336U libraries
  • Patches TFT_eSPI User_Setup.h with FNK0086 config
  • Compiles → build/
  • Flashes bootloader + partitions + app via .venv/bin/esptool.py
./flash-sketch-001-button-beep.sh           # compile + flash
./flash-sketch-001-button-beep.sh --compile  # compile only
./flash-sketch-001-button-beep.sh --flash    # flash only

Step 7 · Toggle Behaviour
👤
Richard

great but it keeps beeping. when i hit the button, start the beeping. when i hit it again, stop the beep

✦
Claude

Redesigned as a toggle. Audio now runs as a FreeRTOS task on Core 0, continuously generating a sine wave into a phase accumulator. The UI loop on Core 1 simply flips a volatile bool beeping.

10ms fade-in/out envelope avoids clicks on start and stop.

  • Tap once → button turns red, shows STOP, 880 Hz tone starts
  • Tap again → button turns blue, shows BEEP, tone stops
// FreeRTOS audio task — Core 0
xTaskCreatePinnedToCore(beep_task, "beep", 4096, NULL, 1, NULL, 0);

// Loop on Core 1 — leading-edge toggle
if (hit && !prev_hit) {
    beeping = !beeping;
    draw_button(beeping);
}
