Simple enough — bouncing rounded rectangle with "DVD" text, float-precision position so motion stays smooth across frame rates. The classic behaviour:

  • Logo travels at velocity (1.7, 1.3) — irrational ratio so it never loops back exactly
  • On wall hit: reflect velocity and advance palette index (skip current colour)
  • Erase old position, draw new — only when pixel coords actually changed

Step 2 · Implementation Details
✦
Claude

The logo is drawn as a filled rounded rectangle (LOGO_W=88, LOGO_H=44, r=8) with a thin inner border for depth, and "DVD" text centred in black — readable against any of the 8 palette colours.

ColourRGB565Hex
Red0xF800#FF0000
Green0x07E0#00FF00
Blue0x001F#0000FF
Yellow0xFFE0#FFFF00
Cyan0x07FF#00FFFF
Magenta0xF81F#FF00FF
Orange0xFD20#FF6400
White0xFFFF#FFFFFF

Colour picks skip the current colour ((ci + 1 + rand() % 7) % 8) so it always visually changes.

// Erase only the previous rect, draw the new one
// Only when integer pixel position changes — no flicker
if (nx != px_prev || ny != py_prev || ci != ci_prev) {
  eraseLogo(px_prev, py_prev);
  drawLogo(nx, ny, PALETTE[ci]);
}
delay(12);  // ~83 fps cap
