Step 2 · Colour Palette
✦
Claude

12 colours chosen to cover the primary/secondary spectrum plus a neutral grey:

BLACK, WHITE, RED, ORANGE, YELLOW, LIME,
DARK GREEN, CYAN, BLUE, PURPLE, MAGENTA, GREY

Selected colour gets a white border ring. The ERASE button toggles eraser mode (paints C_BG = black); while active it turns orange and the palette ring disappears — clear visual mode indicator.

Touch routing separates canvas draws from toolbar taps using a touchStartedInCanvas flag — prevents a finger dragging off the canvas from accidentally tapping a colour.

Step 3 · BMP Save to SD
👤
Richard

how does the BMP save work exactly?

✦
Claude

The canvas stores 30×30 RGB565 values in SRAM. On SAVE, the sketch writes a valid 24-bit BMP directly to the SD card using SD_MMC (pins CMD=38, CLK=39, D0=40).

BMP quirks handled correctly:

  • Bottom-up row order — BMP stores rows from bottom to top, so rows are written in reverse
  • BGR byte order — BMP uses B G R not R G B
  • Row padding — rows padded to 4-byte boundary (30 px × 3 bytes = 90, padded to 92)
  • RGB565 → RGB888 conversion — 5-bit R/B expanded to 8-bit with (v5 << 3) | (v5 >> 2)

Files auto-numbered: canvas_001.bmp, canvas_002.bmp, … Scans SD to find the next free slot. A green SAVED! overlay confirms success; red SAVE FAILED if SD is not mounted.

// Auto-number: find next free canvas_NNN.bmp
for (int i = 1; i <= 999; i++) {
    snprintf(path, sizeof(path), "/canvas_%03d.bmp", i);
    if (!SD_MMC.exists(path)) return i;
}
