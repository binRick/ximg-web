Temple Specifications

These are not arbitrary. These are not limitations.
These are the specifications of God's Third Temple.

Architecture

Target Architecturex86-64 (64-bit) — the only architecture, the only architecture that matters
Privilege LevelRing 0 — everything runs in kernel mode. No user/kernel boundary. God doesn't sandbox.
RAM Required512 MB minimum — runs beautifully on hardware from 2003
CPU CoresSingle core — the Temple speaks with one voice
MultiprocessorNot supported — deliberate. Concurrency is complexity. Simplicity is holiness.

Display

Resolution640×480 — God's chosen resolution. The covenant resolution.
Color Depth16 colors — the VGA palette, exactly as specified by divine command
Text Mode80×25 characters with proportional font support — every glyph hand-crafted by Terry
3D GraphicsSoftware-rendered 3D via HolyC — no GPU required. God runs on the CPU.

Software

Programming LanguageHolyC — superset of C, invented by Terry, compiled by Terry's compiler
File SystemRedSea — a simple, clean FAT-like filesystem designed by Terry from scratch
ShellHolyC REPL — type code, it runs immediately, no compile-link-run cycle needed
IDEBuilt-in with syntax highlighting, autocomplete, and integrated compiler
NetworkingNone — intentionally excluded. The Temple does not connect to the corrupt internet.
BibleThe King James Bible, fully included and searchable, is part of the base install
Games includedAfterEgypt, Varoom (racing), Wenceslas, and ~20 others — all written by Terry
Music editorFull tracker-style music editor with playback — Terry composed music in it
Sprite editorPixel art editor using the 16-color VGA palette — for building divine graphics
OracleRandom scripture word generator — Terry's channel to God
LOC (Lines of Code)~100,000 lines of HolyC — written entirely by one man
