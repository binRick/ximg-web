Terry A. Davis was a computer programmer who built TempleOS — a fully functioning 64-bit operating system — entirely by himself over fifteen years. This was not a toy. It was not a school project. It was a complete, working OS: kernel, compiler, shell, graphics library, file system, debugger, sprite editor, music system, games, and a Bible. Every single line written by one person, in a custom language he invented himself, called HolyC.

Terry believed God commanded him to build it as the modern equivalent of Solomon's Temple — the Third Temple prophesied in scripture. He chose the constraints deliberately: 640×480, 16 colors, ring-0 execution, no networking. These were not limitations. They were a covenant. God's chosen resolution. God's chosen palette. Every pixel placed with divine intention.

The engineering achievement is staggering. Most operating systems are the work of thousands of engineers over decades. TempleOS was one man's mind made executable. It ran real programs. It had a working compiler that compiled itself. It had a file system, a 3D flight simulator, an ANSI art viewer, a music tracker, and a feature where randomized scripture words served as messages from God — the Oracle.

Terry struggled with paranoid schizophrenia. He was often homeless in his final years. He died on August 11, 2018, struck by a train in The Dalles, Oregon. He was 48 years old.

The programming community debated how to feel about him. His output resolved that debate. You can dismiss the man. You cannot dismiss the work.

HolyC — The Language of God

HolyC is a superset of C that Terry designed and implemented from scratch. It supports default arguments, inline assembly, interactive REPL execution, and runs entirely in ring 0 — no user/kernel boundary. Every HolyC program is God's direct expression through silicon.

HelloGod.HC HolyC Compiler  ✓  Ring-0

Key Language Features

Default argumentsU0 Fn(I64 x, I64 y=10) {} — y defaults to 10 if not provided
Immediate executionType any expression in the shell and it runs instantly — no compile step
Inline assemblyasm { MOV RAX, 1; } — seamlessly embedded in HolyC code
Type systemI8/U8, I16/U16, I32/U32, I64/U64, F64 — explicit signed/unsigned sizing
ClassesStructs with methods — no vtables, no inheritance, no bloat. Pure and holy.
No user modeEverything runs in ring 0. The OS is the app. The app is the OS. God doesn't sandbox.
Self-hostingThe HolyC compiler is written in HolyC and compiled by itself
Compiled to memorySource compiles directly to executable code in RAM — no object files, no linker

RECENT TRANSMISSIONS

The 16 Sacred Colors

"God said 640×480, 16 colors is a covenant like circumcision."
— Terry A. Davis

Every TempleOS screen — every pixel of the Temple — uses exactly these 16 colors. Not 256. Not millions. Sixteen. The VGA palette. God's palette. Terry didn't pick them. He received them.

The 16-color constraint was deliberate and divine. Terry argued that more colors would introduce corruption — artistic ego, visual noise, the sin of excess. The 16-color palette forced clarity. Every color choice had to be intentional. The Temple's graphics are blocky, bright, and unmistakably holy because there is no room for ambiguity. God speaks in primary colors.

The Sacred Timeline

December 15, 1969
Terry A. Davis Born
Born in West Allis, Wisconsin. One of ten children. Would go on to study electrical engineering at Arizona State University, graduating with a degree and going to work as an embedded systems engineer at Ticketmaster in the late 1990s.
1996
First Episode
Terry experiences his first psychotic episode, diagnosed with bipolar disorder and later paranoid schizophrenia. He is hospitalized. This would define the rest of his life — the illness and the genius existing simultaneously.
2003
The Divine Command
God commands Terry to build the Third Temple — not a physical structure, but a digital one. A pure operating system, uncorrupted by commercial interests, built to God's specifications: 640×480, 16 colors, a simple architecture where God can speak directly to the programmer.
2005
LoseThos — First Public Release
The OS initially called LoseThos boots for the first time as a publicly viewable project. It already has a working kernel, graphics output, and a primitive shell. Written entirely in a C dialect Terry is developing in parallel with the OS itself.
2008–2012
J Language → HolyC Emerges
Terry iterates through several names and versions of his programming language before settling on HolyC. He adds default arguments, inline assembly, and a self-hosting compiler — meaning HolyC can compile itself. The OS is renamed SparrowOS, then finally TempleOS.
2012
TempleOS 1.0 — The Temple Opens
TempleOS is officially released with its permanent name. The project includes: a working 64-bit kernel, the HolyC compiler, a full IDE with syntax highlighting, a file system (RedSea), a music editor, a sprite editor, a 3D flight simulator, dozens of games, and the Oracle — the divine random-word scripture reader.
2013–2017
Continuous Revelation
Terry continues improving TempleOS while posting prolific videos on YouTube. The programming community slowly takes notice. His YouTube channel amasses hundreds of videos — some brilliant technical demonstrations, some distressing psychotic episodes. He becomes a cult figure.
2017
Homelessness
Terry's living situation deteriorates. He spends periods homeless, living in his car. He continues to appear online, posting videos and writing code. The community that had mocked him begins to feel guilt. Others try to help. He resists.
August 11, 2018
Terry A. Davis Passes
Terry Davis is struck and killed by a train in The Dalles, Oregon. He was 48 years old. The cause was ruled accidental. The programming community mourns. TempleOS lives on — preserved and studied as one of the most remarkable individual software achievements in history.
2018 – Present
The Legacy Endures
TempleOS is studied in universities and by OS developers worldwide. GitHub mirrors preserve the source. YouTubers do deep technical breakdowns. Books are written. Terry's HolyC compiler is analyzed with reverence. The Temple stands forever in silicon, exactly as God specified.
