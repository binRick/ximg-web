proc-trace-tls

Intercept plaintext TLS traffic before it's encrypted — per-process, in real time. Hooks directly into SSL_read / SSL_write via ftrace uprobes. No eBPF. No ptrace. No CA install. Works through cert pinning.

CAPABILITIES
🔓

Plaintext Before Crypto

Hooks SSL_read / SSL_write at the function boundary — you see data exactly as the application sees it, before OpenSSL touches it.

📍

ftrace Uprobes

Uses the kernel's built-in uprobe infrastructure via /sys/kernel/debug/tracing. Available on every Linux distro since kernel 4.17 — no eBPF, no kernel modules, no extra packages.

🎯

Per-Process Filter

Use -p PID[,PID,...] to watch only specific processes, or omit it to catch all TLS calls system-wide.

🔍

Auto Library Detection

Scans /proc/<pid>/maps and common system paths to find libssl.so automatically. Override with -l if needed.

🛡️

Bypasses Cert Pinning

Works regardless of cipher suite or certificate pinning. The uprobe fires on the function, not the network packet — TLS fingerprinting and pinning are irrelevant.

🔌

Zero Dependencies

Single static binary. Requires only debugfs (mounted by default) and root. No BCC tools. No kernel headers. No Python.

WHY THIS EXISTS

See what your app is actually sending over HTTPS

Every HTTPS request your app makes is encrypted. But somewhere between your application code and the network, there's a moment when the data is plaintext — right at the OpenSSL function boundary.

Existing approaches are painful: MitM proxies break cert pinning. Wireshark needs TLS key material. strace works but is slow and verbose. eBPF-based tools like bcc/ssl_sniff require kernel headers and a working BCC install.

proc-trace-tls uses the same kernel mechanism as eBPF uprobes — but accessed via the much simpler ftrace interface that's been in every kernel since 4.17, with no compiler, no headers, and no extra packages.

How it works in one sentence: ftrace uprobes insert a kernel hook at the entry point of SSL_read / SSL_write in libssl.so; proc-trace-tls reads the resulting event stream from /sys/kernel/debug/tracing/trace_pipe.
