Open Source

rbterm

A cross-platform terminal emulator written in C, rendered with raylib. Tabs, embedded SSH, scrollback, truecolor, emoji, mouse selection, session logging, and live palette switching — no Electron, no frameworks.

Features

Tabbed Interface

Up to 16 concurrent shells. Each tab has an independent PTY, scrollback, and selection. Background tabs stay live. Tab labels track the current working directory.

VT500-Style Parser

Full SGR support (16/256/truecolor), cursor movement, scroll regions, alt screen. UTF-8 and wide-character rendering with intelligent reflow on resize.

Mouse Selection

Click-drag to select, double-click for word, triple-click for row. Translucent overlay highlights the selection. Cmd+C to copy, Cmd+V to paste.

Live Font Resize

Cmd++ / Cmd+- / Cmd+0 to grow, shrink, or reset. Reflows all tabs automatically so nothing gets clipped or lost.

Intelligent Reflow

Window resize wraps and unwraps long lines intelligently. Overflow goes to scrollback. Cursor position is preserved after every resize.

Color Emoji (macOS)

Rendered via Core Text + SBIX bitmap fonts. Font substitution handles missing glyphs. Windows falls back to monochrome stub.

OSC Palette Support

Works with the pal CLI tool — switch your terminal's entire color palette with a single command, no restart needed.

Cross-Platform PTY

macOS/Linux use forkpty with non-blocking read. Windows 10+ uses ConPTY with a reader thread and ring buffer. Same binary on every platform.

Embedded SSH

Cmd+Shift+T opens an SSH form with saved hosts from ~/.ssh/config. Key auth via ssh-agent or ~/.ssh/id_*. Trust-on-first-use host keys. Each SSH session is a native tab — no shell in the middle.

Settings Modal

Cmd+, opens settings: live font size adjustment and session logging toggle. Logging writes raw PTY bytes to timestamped files — good for replay and debugging.

Architecture

User Input (raylib) → input_poll() → pty_write() → Shell Process
                                            ↕
                                      stdout of shell
                                            ↓
Screen Buffer (glyph grid) ← screen_feed() (VT parser) ← pty_read()
        ↓
renderer_draw() (raylib output)

Each tab owns a Pty * (pseudo-terminal connection), a Screen * (grid buffer + scrollback), a Selection (mouse selection state), and title/CWD tracking.

Source Layout

src/main.c          Event loop, tab bar UI, SSH modal, settings, clipboard
src/pty.h           Platform-agnostic PTY interface (public)
src/pty_internal.h  Shared struct Pty + backend prototypes
src/pty_dispatch.c  Dispatches pty_* calls to local/SSH backend
src/pty_unix.c      Local PTY via forkpty (macOS/Linux)
src/pty_win.c       Local PTY via ConPTY (Windows)
src/pty_ssh.c       SSH backend via libssh — key auth, TOFU host keys
src/screen.c        ANSI/VT parser, grid buffer, scrollback, reflow
src/render.c        raylib rendering, glyph atlas, emoji cache
src/input.c         Input translation (raylib events → PTY bytes)
src/emoji_mac.m     CoreText emoji rasterizer (macOS)
src/emoji_stub.c    No-op stub (Linux/Windows)
CMakeLists.txt      CMake build (fetches raylib 5.5 automatically)
Makefile            Quick macOS/Linux build

Build

# macOS (fastest path)
brew install raylib libssh
make                # ./rbterm
make app            # ./rbterm.app with icon + Info.plist

# CMake (cross-platform, self-contained)
cmake -S . -B build && cmake --build build

# Linux: also needs libssh-dev (apt install libssh-dev)
